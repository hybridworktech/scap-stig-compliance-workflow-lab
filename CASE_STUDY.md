# SCAP/STIG Compliance Automation Lab Case Study

## Portfolio Positioning

This is a synthetic security compliance automation project. It demonstrates how raw SCAP/STIG-style findings can be transformed into remediation trackers, compliance summaries, dashboard views, and runbooks without exposing real infrastructure data or founder ideas.

## One-Line Summary

Built a synthetic SCAP/STIG compliance automation lab that parses benchmark-style findings, prioritizes CAT I/CAT II risks, generates remediation trackers, and produces before/after compliance reports without exposing sensitive infrastructure data or proprietary product ideas.

## The Problem

Raw compliance scan output can be difficult for engineering teams to act on quickly. Security and infrastructure teams need a repeatable way to:

- Prioritize CAT I, CAT II, and CAT III findings
- Track remediation ownership and status
- Compare before/after scan results
- Generate evidence-ready reports
- Preserve sensitive system details

## What I Built

The lab includes:

- Synthetic SCAP/STIG-style XML scan samples
- PowerShell parser and report engine
- CSV remediation tracker
- Markdown before/after compliance summary
- Static HTML dashboard
- Generated Markdown runbooks for open findings
- Read-only validation check script pack

## Progression Story

```text
Raw synthetic scan data
  -> PowerShell parser
  -> Normalized findings
  -> Remediation tracker
  -> Before/after compliance summary
  -> HTML dashboard
  -> Open-finding runbooks
```

## Privacy-Safe Boundaries

Included:

- Synthetic scan files
- Fake host aliases
- Generic demo control IDs
- Generated reports
- Safe portfolio language

Excluded:

- Real hostnames
- IP addresses
- Usernames
- Client or employer data
- Private scan IDs
- Founder ideas
- Proprietary product concepts
- Production architecture
